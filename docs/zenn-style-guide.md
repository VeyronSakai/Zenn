# Zenn 記事スタイルガイド

このドキュメントは、Zenn 記事を執筆する際のスタイルと規約をまとめたものです。

## 文体

### 基本ルール

- **敬体（です・ます調）** で統一
- 一文は **60文字以内** を目安
- 段落は **3-5文** で区切る
- 専門用語は初出時に説明を補足

### 良い例

```text
fastlane を用いて iOS ビルドを試みた際に私が遭遇した以下の CodeSign のエラーとその解決方法について記載しました。
```

### 避けるべき表現

- 冗長な前置き
- 曖昧な指示語（「これ」「それ」の多用）
- 過度な敬語

## 記事構成

### 標準的な構成

1. **はじめに / 概要**: 記事の目的と対象読者を明示
2. **本題**（複数セクション）: 論理的な順序で解説
3. **まとめ**: 要点の整理
4. **参考**: 参考リンク（任意）

### 見出しレベル

- `##` (H2): メインセクション
- `###` (H3): サブセクション
- `####` (H4): 詳細項目（なるべく避ける）

## Zenn Markdown 記法

### frontmatter

```yaml
---
title: "タイトル（30-60文字）"
emoji: "📝"
type: "tech"  # tech または idea
topics: [topic1, topic2, topic3]  # 1-5個
published: false  # 公開時に true
---
```

### コードブロック

言語を必ず指定する。

````markdown
```typescript
const greeting: string = "Hello, Zenn!";
```
````

### メッセージボックス

```markdown
:::message
通常のメッセージ
:::

:::message alert
警告メッセージ
:::
```

### アコーディオン

```markdown
:::details タイトル
折りたたまれる内容
:::
```

### 画像

```markdown
![代替テキスト](/images/<slug>/image.png)
```

画像は `images/<slug>/` ディレクトリに配置。

## コードスニペット

### ガイドライン

- 言語を明示する
- 適切なコメントを含める
- 実行可能な完全なコードを心がける
- 長いコードは分割して段階的に説明

### 例

```ruby
# fastlane match でキーチェーンパスワードを設定
match(
    type: "appstore",
    keychain_password: ENV["KEYCHAIN_PASS"],  # 環境変数から取得
)
```

## SEO 考慮

### タイトル

- 主要キーワードを含める
- 具体的で検索されやすい表現
- 30〜60 文字が最適

### topics

- 主要技術を必ず含める
- 関連技術を 2〜3 個追加
- 既存の人気タグを優先

## チェックリスト

記事公開前に確認する。

- タイトルが具体的で検索キーワードを含む
- emoji が内容に適切
- topics が 1〜5 個設定されている
- 導入で記事の目的が明確
- コードブロックに言語指定がある
- 画像パスが正しい
- 誤字脱字がない
- まとめで要点が整理されている
- textlint と markdownlint を実行してもエラーが出ないこと
